Welcome to the **Rybrary Command Center**. Where this digital library management system is built for high-stakes organization, allowing its users to deploy (borrow) and recover (return) intellectual assets (Books) with military precision.

## 📝 Mission Overview
This Java application serves as a central hub for a book collection for a library. To ensure maximum readability during field research, I have implemented a **Tactical Monospaced Interface** featuring a Deep Forest Olive Green color and Faded Khaki color palette.

---

## 📁 The Asset Files
To ensure operational security, this project is divided into three core Java source files:
1.  **`Book.java`**: The "Asset Profile." Stores critical data including IDs, ISBNs, and page counts.
2.  **`Inventory.java`**: The "Armory." Manages the deployment logic and generates the **Technical List** for asset review.
3.  **`RyleysLibrarySystem.java`**: The "Command Console." Handles the Swing GUI, button hover effects, and the tactical header.

---

## 🛠 Tactical Enhancements
*   **Persistent Archives:** All data is serialized into `library_data.ser`, ensuring the collection is preserved between missions.
*   **Field Validation:** The system now double-checks all new entries to prevent "Empty Asset" errors.
*   **Hover-Responsive Interface:** Buttons react to user focus, switching colors for better visibility in the field.
*   **Technical Popup:** A scrollable window allows for a full debrief of all available assets including Title, Author, and ID.
